# 《星航棋域：Orbit Dominion》SLG创意玩法设计方案

## 1. 参赛方向与设计定位

本方案对应 ELEX 2026 全球游戏创意设计大赛「方向A / SLG 创意玩法设计」。

参赛要求关注点包括：完整 SLG 游戏创意、4X 核心玩法融合设计、大地图战斗机制、世界观与商业化创意体系，并兼顾玩法新意、与 SLG 产品的融合成熟度、设计完整性、商业可行性和 Demo 呈现效果。

《星航棋域：Orbit Dominion》是一款由经典飞行棋规则演化而来的赛季制星际 SLG。它保留飞行棋中“基地起飞、航线跳跃、撞击击毁、终点夺魁”的大众认知，但将核心驱动力从随机掷骰转为行动力、侦察、基地建设、联盟争夺和大地图战术博弈。玩家不再只是等待骰子结果，而是在战争迷雾、航线节点、舰队编组、资源消耗和联盟协作中做出连续策略选择。

一句话概括：

> 用飞行棋的低门槛记忆点，包装一套具备探索、扩张、开发、消灭四大 4X 循环的现代 SLG 赛季玩法。

## 2. 核心创意

传统飞行棋的趣味来自“未知步数”和“踩子击回”。但 SLG 用户需要长期目标、策略深度、社交冲突和可持续成长。因此本方案将飞行棋的核心元素进行如下重构：

| 飞行棋元素 | SLG化改造 | 设计价值 |
| --- | --- | --- |
| 四色阵营 | 四大星盟阵营与联盟体系 | 天然形成阵营认同和 GvG 结构 |
| 基地/停机坪 | 玩家主城和星港 | 承载建造、科技、养成、生产 |
| 起飞 | 舰队出征 | 用行动力和编队替代掷骰 |
| 航线格子 | 六边形星域大地图 | 支撑探索、占领、行军、封锁 |
| 同色跳跃 | 战略航道/折跃门 | 形成关键节点争夺 |
| 撞击击毁 | 舰队接触战和空域拦截 | 从简单惩罚升级为战术战斗 |
| 终点 | 中央奇观“星门核心” | 赛季终局目标，驱动联盟战争 |

核心体验目标是“易懂但不浅”。玩家一眼能理解这是飞行棋灵感的星际争夺，但真正胜负来自侦察、路线选择、兵种克制、资源周转、联盟调度和赛季节奏管理。

## 3. 世界观设定

### 3.1 背景

在人类进入深空殖民纪元后，旧星际航道因一次“星门坍缩”被切断。四个新兴星盟在破碎星域中争夺残存航道、能源矿脉和中央星门核心。传说谁能重启中央星门，谁就能掌握通往全银河殖民网络的最高权限。

四大阵营分别对应飞行棋四色记忆：

| 阵营 | 色彩 | 风格定位 | 玩法倾向 |
| --- | --- | --- | --- |
| 赤曜联邦 | 红 | 工业军团、重火力 | 进攻和攻城加成 |
| 苍穹议会 | 蓝 | 科研城邦、情报网络 | 侦察和科技加速 |
| 翡翠同盟 | 绿 | 生态殖民、资源循环 | 采集和恢复能力 |
| 金辉商约 | 黄 | 商业舰队、雇佣兵 | 贸易和机动调度 |

### 3.2 视觉基调

整体美术采用“高识别色块 + 星图沙盘 + 科幻军事 UI”。大地图以深空星域、六边形网格、航道光轨、星门遗迹和漂浮城市构成。单位设计保留“飞机/棋子”的剪影记忆，但表现为星际战机、预警舰、轰炸艇和航母平台。

## 4. 核心玩法循环

玩家每天的基础循环如下：

1. 侦察未知星域，解除战争迷雾。
2. 派遣舰队占领资源点、航道节点和中立据点。
3. 将资源运回主城，升级建筑、科技和舰队。
4. 与敌对玩家在关键航线上发生拦截、伏击或集结战。
5. 通过联盟协作推进阵营边界，最终争夺中央星门核心。

长期循环以赛季为单位：

1. 赛季初期：地图探索、联盟建立、资源发育。
2. 赛季中期：航道争夺、据点推进、跨区冲突。
3. 赛季后期：中央星门开放，联盟集结争夺奇观。
4. 赛季结算：按照占领时长、联盟积分、个人贡献发放奖励。

## 5. 4X 玩法融合设计

### 5.1 eXplore 探索

地图初始被战争迷雾覆盖。玩家需要派出侦察机点亮星域，发现资源矿、废弃星港、中立舰队、折跃门和敌方基地。

探索不只是开图，而是带有风险和决策：

- 侦察机速度快、成本低，但遇到敌方巡逻舰可能被击落。
- 雷达站等级决定可见范围和敌军行军预警时间。
- 特殊星云会隐藏舰队轨迹，适合埋伏和偷袭。
- 探索遗迹可触发随机事件，例如科技碎片、NPC 伏击、临时增益或星域灾害。

### 5.2 eXpand 扩张

玩家通过占领六边形地块逐步扩张势力范围。占领不是简单铺地，而是围绕“航线连通性”展开。

关键机制：

- 地块必须与己方基地、联盟领地或已控制航道相连，避免无脑飞地。
- 折跃门是高级战略节点，占领后可降低远距离行军时间。
- 航线中继站可作为临时补给点，影响舰队续航。
- 联盟可修建前线星港，用于集结和快速支援。

扩张目标从“占更多格子”升级为“控制更优路线”。

### 5.3 eXploit 开发

资源体系包括三类基础资源和一类战略资源：

| 资源 | 用途 |
| --- | --- |
| 合金 | 建筑升级、舰船生产 |
| 晶能 | 科技研究、折跃消耗 |
| 燃料 | 行军、侦察、战斗行动力 |
| 星核 | 高级科技、赛季奇观争夺 |

主城建筑：

| 建筑 | 功能 |
| --- | --- |
| 星港指挥部 | 解锁建筑等级、舰队上限 |
| 资源精炼厂 | 提升合金和晶能产出 |
| 燃料反应堆 | 提升行动力恢复速度 |
| 雷达站 | 增加视野和预警范围 |
| 兵工厂 | 解锁兵种和强化模块 |
| 战术学院 | 提升编队容量和将领技能 |
| 联盟中枢 | 开启联盟援建、集结和科技 |

资源平衡可用动态模型描述：

```text
dR/dt = Pbase + Ptile + Palliance - Cbuild - Cwar - Ctrade
```

其中 R 为资源存量，Pbase 为主城产出，Ptile 为地块产出，Palliance 为联盟增益，Cbuild 为建设消耗，Cwar 为战争损耗，Ctrade 为市场交换损耗。设计目标是让和平发育、战争扩张和联盟协作之间形成可感知的机会成本。

### 5.4 eXterminate 消灭

战斗从“踩到对方棋子即击回”升级为大地图接触战。舰队在同一格、相邻拦截范围或航道节点相遇时触发战斗。

战斗结算考虑：

- 兵种克制
- 科技等级
- 指挥官技能
- 地形加成
- 士气状态
- 补给距离
- 联盟增援

基础公式：

```text
实际战力 = 舰队基础战力 * 科技系数 * 克制系数 * 地形系数 * 士气系数 * 补给系数
```

战败方不是直接永久消失，而是根据损伤比例进入三种状态：

| 状态 | 结果 |
| --- | --- |
| 轻损 | 返回最近补给点，短时间维修 |
| 重损 | 回主城维修，损失部分兵力 |
| 击毁 | 单位报废，返还少量残骸资源 |

这样既保留飞行棋“被撞回家”的爽点，又符合 SLG 对战损、恢复和资源消耗的长期需求。

## 6. 单位与兵种设计

每名玩家初期拥有四支核心舰队，对应飞行棋的“四枚棋子”，但每支舰队可发展为不同职能。

| 单位 | 定位 | 优势 | 弱点 |
| --- | --- | --- | --- |
| 截击机群 | 高速拦截 | 克制轰炸艇，适合堵航线 | 正面攻城弱 |
| 轰炸艇队 | 攻坚破坏 | 对基地、据点和资源点伤害高 | 机动慢，怕截击 |
| 预警舰 | 视野辅助 | 扩大视野，提供命中和预警 Buff | 直接战斗力低 |
| 护航舰 | 防御支援 | 降低友军战损，保护运输 | 输出不足 |
| 工程舰 | 建设占领 | 修建中继站、采集高级资源 | 容易被突袭 |
| 隐形侦察机 | 潜入侦察 | 可穿越迷雾和星云 | 被雷达反制 |

兵种克制关系：

```text
截击机 > 轰炸艇 > 据点防御 > 护航舰 > 截击机
预警舰和工程舰不参与克制闭环，但提供战略价值。
```

## 7. 大地图战斗机制

### 7.1 六边形星域地图

地图由六边形格组成，地块类型包括：

| 地块 | 效果 |
| --- | --- |
| 普通星域 | 常规移动和战斗 |
| 能源矿带 | 提供晶能和星核 |
| 陨石带 | 降低移动速度，提高防守方闪避 |
| 星云区 | 隐藏行军路线，降低命中 |
| 折跃门 | 长距离跳跃节点 |
| 废弃星港 | 占领后作为前线补给点 |
| 中央星门 | 赛季最终奇观 |

### 7.2 航线与跳跃

“同色跳跃”被改造成战略航线系统。

- 地图中存在天然航道，沿航道移动消耗更低。
- 控制折跃门可进行远距离投送。
- 折跃门需要持续供能，联盟可集资维持。
- 敌方可攻击折跃门使其暂时失效。

因此，航线不是单纯加速点，而是联盟战略命脉。

### 7.3 拦截与封锁

舰队可进入“巡逻”状态，对周围 1-2 格形成拦截圈。敌方经过时会触发拦截判定。

这使玩家可以用较少兵力实现：

- 堵住资源运输线
- 防守折跃门入口
- 掩护联盟集结
- 伏击高价值工程舰和运输舰

## 8. 动态事件与智能 NPC

为了提高大地图变化和观赏性，地图周期性生成动态事件：

| 事件 | 效果 |
| --- | --- |
| 离子风暴 | 行军消耗翻倍，雷达范围降低 |
| 星核潮汐 | 高级资源点短期开启，吸引争夺 |
| 海盗迁徙 | 中立 NPC 攻击高收益资源区 |
| 航道坍缩 | 部分航线暂时不可用，迫使改道 |
| 遗迹苏醒 | 刷新高难度 PVE 据点 |

NPC 阵营采用行为树或轻量 AI 规则：

- 玩家频繁采集某区域，海盗仇恨提高。
- 玩家可用资源贿赂海盗攻击敌方矿点。
- 中立商队会沿航道移动，护送成功可获得联盟积分。
- 高等级 NPC 会优先攻击扩张过快但防守薄弱的玩家。

这让战场不只是玩家之间的静态对抗，而是持续变化的沙盘。

## 9. 联盟与 GvG 设计

### 9.1 联盟目标

联盟是游戏核心社交结构。个人玩家负责成长和局部战斗，联盟负责战略推进。

联盟能力包括：

- 联盟科技
- 领地连通
- 集结进攻
- 援军驻防
- 前线星港建设
- 折跃门维护
- 奇观争夺报名

### 9.2 中央奇观

赛季后期，地图中央的“星门核心”开启。联盟需要占领周边四个控制塔，才能对中央星门发起主权争夺。

胜利条件：

- 中央星门累计占领时长最高的联盟获得赛季冠军。
- 若多联盟占领时长接近，则比较控制塔积分、击毁积分和资源贡献。
- 个人奖励按联盟排名与个人贡献双重结算。

### 9.3 飞行棋情怀表达

中央星门可设计为“终点圆环”。四个阵营从地图四角出发，沿不同航道向中心推进，形成对飞行棋“从基地飞向终点”的视觉呼应。

## 10. 商业化设计

商业化原则是“不卖直接胜利，卖效率、外观、便利和长期目标”。

### 10.1 付费点

| 类型 | 内容 |
| --- | --- |
| 月卡 | 每日资源、加速道具、额外侦察次数 |
| 战令 | 赛季任务、限定舰队皮肤、指挥官碎片 |
| 外观 | 主城皮肤、航线特效、舰队拖尾、折跃动画 |
| 成长礼包 | 新手建筑队列、低阶资源包、通用加速 |
| 联盟礼包 | 集结战参与礼包、联盟建设贡献礼包 |

### 10.2 控制付费破坏性

- PVP 战损上限和医院容量保护中低付费玩家。
- 高级资源主要来自地图争夺，不能完全靠购买替代。
- 付费加速存在日限，避免赛季初被拉开不可追差距。
- 外观和便利内容占主要长期付费比例。

### 10.3 出海适配

ELEX 面向全球市场，本方案天然适合多地区发行：

- 科幻题材文化门槛低，避免强地域历史包袱。
- 四色阵营识别清晰，适合全球用户快速理解。
- 飞行棋/竞速棋类在多地区有类似认知，可降低教学成本。
- 联盟战、赛季制、战令和外观付费符合成熟 SLG 商业模型。

## 11. 新手体验设计

新手前 30 分钟目标是让玩家理解“这不是传统飞行棋，但灵感来自飞行棋”。

新手流程：

1. 建立星港，获得四支初始舰队。
2. 派侦察机点亮第一片迷雾。
3. 占领第一个燃料矿。
4. 使用航道加速完成一次“跳跃”。
5. 与 NPC 海盗发生第一次拦截战。
6. 修复舰队，升级雷达站。
7. 加入联盟，参与一次小型据点集结。

教学重点放在操作中，不使用过多文本解释。每一步都对应一个清晰反馈：开图、占点、跳跃、战斗、修复、联盟协作。

## 12. Demo 实现方案

参赛提交建议采用“完整路演 PPT + Demo 或 2-10 分钟录屏”的组合。Demo 重点展示核心创新，不追求完整商业产品。

### 12.1 Demo 目标

Demo 应展示以下内容：

- 六边形大地图和战争迷雾
- 主城、资源点、折跃门、中央星门
- 四类舰队的移动和占领
- 行动力替代骰子的策略调度
- 航道跳跃和拦截战斗
- 简化联盟争夺中央奇观

### 12.2 推荐技术栈

如果以网页 Demo 形式实现：

| 模块 | 技术 |
| --- | --- |
| 前端 | React / Vue / 原生 Canvas |
| 地图渲染 | Canvas 2D 或 PixiJS |
| 状态管理 | Zustand / Pinia / 简单 JS Store |
| 战斗结算 | TypeScript 规则模块 |
| 数据配置 | JSON 配表 |
| 演示部署 | 本地网页或 Vercel |

如果时间有限，可用 Unity 制作更直观的可视化 Demo，重点是拖拽舰队、占领节点和播放战斗反馈。

### 12.3 最小可行 Demo 范围

建议控制在一张 15x15 六边形地图内：

- 4 个玩家基地，位于四角。
- 12 个普通资源点。
- 4 个折跃门。
- 1 个中央星门。
- 3 类单位：截击机、轰炸艇、预警舰。
- 2 种地形：星云、陨石带。
- 1 套简化战斗公式。
- 1 个赛季倒计时和积分面板。

Demo 的核心交互：

1. 点击己方单位。
2. 显示可移动范围。
3. 点击目标格移动。
4. 若进入航道节点，触发跳跃选择。
5. 若遇敌，弹出战斗结算。
6. 占领中央星门后积分开始增长。

### 12.5 战斗伪代码

```text
attackerPower = attacker.basePower
  * techMultiplier
  * counterMultiplier
  * terrainMultiplier
  * moraleMultiplier

defenderPower = defender.basePower
  * techMultiplier
  * counterMultiplier
  * terrainMultiplier
  * supplyMultiplier

winRate = attackerPower / (attackerPower + defenderPower)

if random() < winRate:
  attacker wins
  defender enters damaged or destroyed state
else:
  defender wins
  attacker returns to nearest supply point
```

为减少随机性，可在 Demo 中将随机区间控制在 10%-15%，保证主要结果由战力、克制和地形决定。

建议 PPT 控制在 15-18 页：

## 15. 风险与优化方向

### 15.1 风险

- 飞行棋元素过弱，可能看不出创意来源。
- 系统过重，新手理解成本过高。
- 大地图航线过强，可能导致强联盟滚雪球。
- 动态 NPC 若过于复杂，Demo 难以实现。

### 15.2 解决方案

- 在视觉和术语上保留基地起飞、四色阵营、航线跳跃、中央终点。
- 新手期先开放小地图和少量单位，逐步解锁 SLG 系统。
- 折跃门设置维护成本和可破坏状态，避免永久垄断。
- Demo 阶段用规则 AI 替代复杂 LLM，保证可控展示。

## 16. 最终卖点总结

《星航棋域：Orbit Dominion》的核心价值在于，它不是简单把飞行棋换皮成科幻题材，而是把飞行棋中最有辨识度的规则转化为 SLG 的战略结构：

- “起飞”变成舰队出征。
- “格子”变成六边形星域。
- “跳跃”变成折跃门和战略航线。
- “撞击”变成拦截与战损结算。
- “终点”变成中央奇观争夺。
- “四色玩家”变成阵营与联盟战争。

这套设计既有大众可理解的创意来源，又具备出海 SLG 所需要的长线养成、联盟社交、赛季竞争和商业化空间。作为参赛作品，它可以用 PPT 完整呈现创意闭环，并通过小型 Demo 展示核心玩法亮点，符合 ELEX 2026 对 SLG 创意玩法设计的方向要求。
